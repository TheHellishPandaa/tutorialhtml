¿Para que no se debe utilizar una tabla?


                Las tables HTML están pensadas para utilizarse con datos tabulados. Por desgracia, mucha gente
                utiliza las tablas HTML para confeccionar la estructura de las páginas web. Por ejemplo, una file para

                contener la cabecera, una fila para contener las columnas de contenido, una fila para contener el pie de
                página, etc. Las razones por las que no se debe usar una tabla como elemento de diseño de la
                estructura de la página son:
                

                1.Las tablas de diseño reducen la accesibilidad para los usuarios con discapacidad visual: Los
                lectores de pantalla que utilizan las personas con visibilidad regucida interpretan las etiquetas de
                una página HTML y leen su contenido para el usuario.
                

                2.Las tablas generan estructuras incorrectas: Los diseños de tabla suelen involuerer estructuras
                de marcado más complejas que las técnicas de diseño. Esto puede dificultar la escritura, el
                mantenimiento y la depuración del código
                

                3.Las tablas no tienen respuesta adaptativa automática: Cuando usas contenedores de diseño
                adecuados (tales como <neader> <section> <articie> y
                 <div>) su ancho predeterminado es el
                100% de su elemento padre. En cambio, las tablas se dimensionan de forma predeterminada
                según su contenido, por lo que se necesitan medidas adicionales para que el diseño de la
                tabla
                funcione de manera efectiva en todos los dispositivos.
                

                
Creacion de una tabla

                

                El siguiente código html permite crear una tabla básica:
                

                
                    
        <table>
        <tr>
    <td>Compañía</td>
    <td>Contacto<td>
    <td>Germany</td>
        </tr>
        <tr>
    <td>Alfreds Futterkiste</td>
    <td>Maria Anders</td>
    <td>Pais</td>
        </tr>
        <tr>
    <td>Cantro comercial Moctezuma</td>
    <td>Francisco Chang</td>
    <td>Méjico</td>
        <tr>
        <table>
                    
                    



                
El contenido de la tabla viene delimitado por las etiquetas <table> y ≪/table>

                
El contenedor más pequeño dentro de una tabla es la celda. El contenido de una celda viene delimitado
                    por las etiquetas: <td> y </td>

                
Las celdas las agrupamos en filas. Cada fila viene delittada por las etiquetas: <tr> y
                    </tr>

                
Si no usamos nada de CSS el aspecto de nuestra tabla es muy pobre. De hecho no se parece a la
                    imagen que todos tenemos de una tabla, en la que las celdas están delimitadas por un borde.

Celdas que abarcan varias filas y columnas

                
A veces queremos que las celdas abarquen varias filas o columnas. Por ejemplo en vuestra tabla
                    horario, la primera fila (en la que aparece el nombre del grupo) está formada por una celda que
                    abarca todas las columnas de una fila. Puedes conseguir ese comportamiento definiendo esa celda así:
                

                
    
<th colspan="6"> VS1ASIR>
    
    

                
Si quieres que una celda abarque varias filas como por ejemplo para visualizar que el lunes las dos
                    primeras clases que tenéis son del módulo de Gestión de Base de Datos, usaríamos el atributo rowspan
                    indicando el número de filas que queremos que ocupe la celda.

                
    
<th rowspan="2"> Ledes Pé, MaIr <br/>(B-001)</th>

Conceptos avanzados y accesibilidad

                
En el segundo artículo de este módulo, analizamos algunas características más avanzadas de las tablas
                    HTML, como los subtítulos/resúmenes, la agrupación de filas en las secciones del encabezado, el
                    cuerpo y el pie de página; y también analizamos la accesibilidad de las tablas para usuarios con
                    discapacidad visual.

                
Añadir un subtítulo a la tabla

                
Puedes dar un título a tu tabla colocándolo dentro de un elemento
                        <caption> y anidándolo dentro del elemento
                        <table>. Debes ponerlo justo debajo de la etiqueta de apertura <table>
                
Añadir estructura a una tabla con las etiquetas <thead>
                        <tbody> <tfoot>

                
La etiqueta <thead> nos permite marcar el encabezado de una
                    tabla, <tfoot> marca el pie de la tabla y
                        <tbody> delimita el cuerpo de la misma

                
Estos elementos no mejoran las características de accesibilidad de la tabla para los usuarios de
                    lectores de pantalla ni su aspecto visual en sí. Sin embargo, son muy útiles para la aplicación de
                    estilo y la compaginación, porque actúan como soportes útiles para añadir CSS a tu tabla.

                
Para usarlos ten en cuenta lo siguiente:

                
    
                    
  • El elemento  <thead> delimita el encabezado de la tabla; esta suele ser la primera
                        fila, que contiene los encabezados de las columnas, pero no siempre es así. Si utilizas los
                        elementos <col>/<colgroup> , el encabezado de la tabla debe estar justo debajo.
    • 
                    
  • El elemento <tfoot> la parte de la tabla correspondiente al pie de página; esta
                        podría ser una fila final con elementos en las filas anteriores. Puedes incluir el pie de página
                        de la tabla justo en la parte inferior de la tabla, donde esperarías que esté, o justo debajo
                        del encabezado (y el navegador lo mostrará aun así en la parte inferior de la tabla).
    • 
                    
  •  El elemento <tbody> delimita las otras partes del contenido de la tabla que no están en
                        el encabezado o en el pie de página de la tabla. Aparecerá debajo del encabezado de la tabla o,
                        a veces, en el pie de página, según cómo hayas decidido
    •
